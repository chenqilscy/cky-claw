# 问题清单

> **根因分析（2026-04-14）：** 所有 500 错误均为环境问题，非代码 Bug。
> - 数据库连接：PostgreSQL 需要正常运行（Docker 或本地服务）
> - JWT 密钥：后端 `CKYCLAW_SECRET_KEY` 必须与签发 JWT 时一致，否则令牌校验失败
> - 已验证：所有 API 端点（agents、environments、guardrails、tool-groups、providers）在后端正常启动后均返回 200
> - Provider 创建 API 也正常工作（POST `/api/v1/providers`）
>
> **解决步骤：**
> 1. 确保数据库正在运行（`docker-compose up -d db redis` 或本地 PostgreSQL 服务）
> 2. 确认 `.env` 中 `CKYCLAW_DATABASE_URL` 指向可达的数据库
> 3. 执行迁移：`cd backend && uv run alembic upgrade head`
> 4. 启动后端：`cd backend && uv run uvicorn app.main:app --reload`
> 5. 启动前端：`cd frontend && pnpm dev`

# 功能特性升级
## 工具组
> **回答：**
> - **工具组功能定义**：ToolGroup 是一组相关工具的逻辑分组（如 web-search、code-executor、file-ops 等），由 `ckyclaw_framework/tools/` 中的 `ToolGroup` + `ToolRegistry` 实现。它不等同于 MCP 工具，但 MCP 工具可以注册到工具组中。
> - **系统 Agent Skill**：Skill 是独立于工具组的概念，由 E1 Skill Factory 实现（Agent 自主创建技能 + AST 白名单安全 + 持久化）。
> - **工具安装/升级/卸载**：当前通过配置管理（CRUD API），MCP 工具通过 `mcp/` 模块的 stdio/sse/http 三种传输方式集成。
> - **动态加载**：MCP 工具和 ToolGroup 均支持运行时动态注册，Runner 在每次 Agent Loop 开始时从 ToolRegistry 解析工具列表。
