  1. 架构风格对比

  ┌─────────────────────┐                ┌─────────────────────┐
  │  CrabBoard (React)  │                │  frontend (React)   │
  │  WS + REST          │                │  REST + fetch       │
  ├─────────────────────┤                ├─────────────────────┤
  │  nextcrab-api-v2    │                │  backend (FastAPI)  │
  │  axum + WS          │                │  uvicorn + REST     │

  3. 执行引擎对比（最核心差异）

  Runner vs EngineLoop

  ┌──────────────┬───────────────────────────────────┬───────────────────────────────────────────────┐
  │     维度     │      NextCrab engine_loop.rs      │               CkyClaw runner.py               │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 语言         │ Rust (tokio)                      │ Python (asyncio)                              │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 入口         │ run_loop()                        │ Runner.run() / run_sync() / run_streamed()    │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 默认最大迭代 │ 25                                │ max_turns（可配置）                           │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 工具执行     │ 分区：并发安全并行 + 串行工具排队 │ 全部并行 (asyncio.TaskGroup)，可选 Semaphore  │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 工具超时     │ 全局 30s/300s                     │ FunctionTool.timeout / RunConfig.tool_timeout │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 取消机制     │ CancellationToken 传播            │ 无原生取消（靠 max_turns）                    │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 动态工具     │ Deferred Tool（tool_search 激活） │ ToolSearchTool（>20 工具时搜索）              │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 重试         │ Circuit Breaker + Fallback Chain  │ 指数退避，单 provider                         │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 上下文窗口   │ 128K 默认，5 层压缩管道           │ HistoryTrimmer（滑动窗口/Token 预算）         │
  ├──────────────┼───────────────────────────────────┼───────────────────────────────────────────────┤
  │ 错误处理     │ 所有错误捕获为工具结果            │ 所有异常捕获为错误字符串                      │
  └──────────────┴───────────────────────────────────┴───────────────────────────────────────────────┘

  上下文管理对比（关键差异）

  ┌─────────┬─────────────────────────────────────────┬───────────────────────────────────────────────┐
  │  层级   │             NextCrab (5 层)             │               CkyClaw (2 策略)                │
  ├─────────┼─────────────────────────────────────────┼───────────────────────────────────────────────┤
  │ Layer 0 │ Snip（截断旧消息）                      │ —                                             │
  ├─────────┼─────────────────────────────────────────┼───────────────────────────────────────────────┤
  │ Layer 1 │ ToolResultBudget（截断大工具输出）      │ —                                             │
  ├─────────┼─────────────────────────────────────────┼───────────────────────────────────────────────┤
  │ Layer 2 │ Context Collapse（消息折叠）            │ —                                             │
  ├─────────┼─────────────────────────────────────────┼───────────────────────────────────────────────┤
  │ Layer 3 │ AutoCompact（LLM 摘要压缩）             │ —                                             │
  ├─────────┼─────────────────────────────────────────┼───────────────────────────────────────────────┤
  │ Layer 4 │ ReactiveCompact（API 返回超长时再压缩） │ —                                             │
  ├─────────┼─────────────────────────────────────────┼───────────────────────────────────────────────┤
  │ 等价物  │ —                                       │ HistoryTrimmer：SLIDING_WINDOW / TOKEN_BUDGET │
  └─────────┴─────────────────────────────────────────┴───────────────────────────────────────────────┘

  NextCrab 的 5 层压缩管道远比 CkyClaw 精细，按代价递增逐步处理，且有 ReactiveCompact 作为安全网。
